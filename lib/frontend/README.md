# Frontend - SvelteKit v5 UI

Demostrate AI-LinkMO exploration

## Quick Start

```bash
npm install
npm run dev
```

## Key files

### **`src/lib/constants.ts`**

**Main configuration file.** Customize application behaviour here:

- Application metadata (name, logos, URLs)
- API endpoints, UI Accordian groups, and backend intergration
- UI dimensions and layout parameters
- UI wanted filters per Endpoint (filed names from filters.svelte.ts)
- UI Personas (demo users)

### **`src/lib/states/filters.svelte.ts`**

**Reactive filter state management** using Svelte 5 runes:

- Maintain search and filter parameters
- Dropdown options generation
- Filter validation and application logic
- Extend functionality by adding new filter fields here
- Aligned to `/lib/api/OpenApi.yaml` as single source of truth

## Configuration Files

- `vite.config.js` - Vite bundler, proxy API to backend
- `svelte.config.js` - SvelteKit configuration
- `tailwind.config.js` - Tailwind CSS customization
- `tsconfig.json` - Typescript compiler options
- `package.json` - Dependencies and scripts

## Extending Functionality

1. **Add new entityt type:**
   - Update `ENDPOINTS`, `ENDPOINT_GROUPS` in `constants.ts`

2. **Customize UI:**
   - Modify layout dimensions in `constants.ts`
   - Edit component files in `components/ui/`
   - Adjust global styles in `app.cs`
   - Adjust component styles in specific css files.

3. **Add filters**
   - Update `DROPDOWN_FIELDS` in `filters.svelte.ts`
   - Add filter parameters in `filters.svelte.ts`
   - Add filter Dropdown in `filters.svelte.ts`
   - Enable UI visibility via `UI_WANTED_FILTERS` in `constants.ts`
