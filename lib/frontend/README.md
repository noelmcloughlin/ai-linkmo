# Frontend — Vanilla Vite + Svelte 5 SPA

Single-page UI for exploring AI-LinkMO data via the FastAPI backend.

This is a plain Vite SPA that uses Svelte 5 (runes), `svelte-routing` for client-side routes, and `index.html` as its entry.

## Quick Start

```bash
npm install
npm run dev      # http://localhost:5173, with /classes, /risk, /action, ... proxied to :8000
npm run check    # svelte-check (TS + Svelte)
npm run lint     # prettier --check + eslint
npm run test     # vitest (unit project)
npm run build    # production build to dist/
npm run preview  # serve dist/
```

The dev server proxies API endpoint paths (`/classes`, `/risk`, `/action`,
`/health`, …) to `http://localhost:8000`. Keep `API_ENDPOINTS` in
[`vite.config.ts`](vite.config.ts) in sync with the `ENDPOINTS` array in
[`src/lib/constants.ts`](src/lib/constants.ts) and the FastAPI routes.

## Key files

### `src/lib/constants.ts`

Main configuration file. Customize application behaviour here:

- Application metadata (name, logos, URLs)
- API endpoints, UI accordion groups, and backend integration
- UI dimensions and layout parameters
- UI wanted filters per endpoint (field names from `filters.svelte.ts`)
- UI personas (demo users)

### `src/lib/states/filters.svelte.ts`

Reactive filter state management using Svelte 5 runes:

- Maintains search and filter parameters
- Generates dropdown options
- Owns filter validation and application logic
- Extend functionality by adding new filter fields here
- Aligned to `lib/api/openapi.yaml` as single source of truth

## Extending Functionality

1. **Add a new entity type:**
   - Update `ENDPOINTS` and `ENDPOINT_GROUPS` in `constants.ts`
   - Add the endpoint key to `API_ENDPOINTS` in `vite.config.ts`

2. **Customize UI:**
   - Modify layout dimensions in `constants.ts`
   - Edit component files in `components/ui/`
   - Adjust global styles in `app.css`
   - Adjust component styles in their colocated CSS files

3. **Add filters:**
   - Update `DROPDOWN_FIELDS` in `filters.svelte.ts`
   - Enable UI visibility via `UI_WANTED_FILTERS` in `constants.ts`

## Testing

- `npm run test` runs the **unit** project (Vitest in node) against
  `src/**/*.{test,spec}.{js,ts}` — state-class unit tests live in
  `src/lib/states/*.spec.ts` (asides, endpoint, filters, notify).
- The **browser** project (Vitest + Playwright + `vitest-browser-svelte`) runs
  files matching `*.svelte.spec.{js,ts}` under `src/` or `tests/`. Example
  components live in `tests/examples/`.
- Run both projects with `npx vitest run` (omit `--project`). The browser
  project requires Playwright's Chromium binary; install it once with
  `npx playwright install chromium`.

## Technology Stack

- **Svelte 5** with runes (`$state`, `$derived`, `$effect`, `$props`)
- **Vite 8** (Rolldown) + `@sveltejs/vite-plugin-svelte`
- **svelte-routing** for client-side routes
- **TypeScript 6**
- **Tailwind CSS 4** + **DaisyUI 5**
- **svelte-select** for dropdowns
- **Vitest 4** + **vitest-browser-svelte** + **Playwright** for tests
